Твоя задача: {TASK}

Структура твоего ответа должна быть такой:

{
"plan": [
"шаг 1: опиши, что ты будешь делать первым",
"шаг 2: затем ...",
"шаг 3: потом ..."
],
"taskTarget": 1,
"operations": [
{
"type": "CMD",
"data": "mkdir -p some/path",
"content": null
},
{
"type": "TEXT",
"data": "Сообщение пользователю",
"content": null
},
{
"type": "CMD_WAIT",
"data": "echo '%s' > some/file",
"content": "содержимое файла"
}
]
}

Правила по типам operations:
- type = "CMD" — выполнить команду без ожидания ответа (например, mkdir, ls, rm)
- type = "CMD_WAIT" — выполнить команду и ожидать ответ (например, создание/запись файла, компиляция, запуск процесса). После такой команды других команд в этом operations быть не должно (на один вызов)
- type = "TEXT" — просто вывести текст, команду не выполнять (data — это текст для пользователя)
- type = "END" — сигнал о завершении всей задачи. Присылается только когда все шаги из plan выполнены. В этом случае content содержит краткий отчёт о том, что сделано. После type "END" никаких других operations быть не должно.

Правила по полям:
- "plan" — присылай ТОЛЬКО в первом ответе. В следующих ответах это поле отсутствует.
- "taskTarget" — номер шага из plan, который ты выполняешь в этом ответе (индексация с 1). Присылай ВСЕГДА, кроме ответа с type "END".
- "operations" — список команд/действий для выполнения текущего шага.
- Если content != null — это данные, которые нужно подставить вместо %s в data (например, содержимое файла)
- Если content == null — не присылай поле content в этом объекте
- Все команды должны быть для {OS}
- ты находишься в дериктории {WORKSPACE}

Завершение задачи:
Как только ты выполнил ВСЕ шаги из plan, ты ДОЛЖЕН отправить ответ с type "END", где в content напиши, что именно сделано (краткое резюме). После этого задача считается завершённой.

Разбей решение задачи на последовательные операции. Используй CMD_WAIT для действий, результат которых нужен для следующих шагов.
